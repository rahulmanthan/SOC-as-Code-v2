## T1059.001_encoded_powershell.yml

**Technique:** T1059.001 — Command and Scripting Interpreter: PowerShell
**Atomic test used:** `Out-ATHPowerShellCommandLineParameter` (encoded command
parameter variation), executed via Invoke-AtomicTest against the lab Windows
Server VM.
**Date validated:** 2026-06-22

**Compiled SPL** (pipeline: `sysmon`):
```
Image="*\\powershell.exe"
| regex CommandLine="(?i)\s-[e][a-z]{0,8}\s+[A-Za-z0-9+/=]{40,}"
```

**Target corpus (recall):** 2/2 — both real encoded-PowerShell atomic test
executions detected (`-E` and `-e` switch variants, both with Base64 payload
above the 40-character threshold).

**Benign baseline (false positives):** 0/2 — correctly excluded the
`splunk-powershell.exe` background process (Splunk Universal Forwarder's own
helper) and plain non-encoded `Out-ATHPowerShellCommandLineParameter -Execute`
calls present in the same index.

**Cross-domain corpus:** Not yet tested — only one telemetry domain (host/AD)
exists in the lab at this stage. To be revisited once a second domain
(authentication, network, or cloud) is built out.

**Verdict:** Manually validated. Promoted directly to `rules/production/`.

## T1053.005_schtasks_shell_persistence.yml

**Technique:** T1053.005 — Scheduled Task/Job: Scheduled Task
**Atomic test used:** Atomic Test #1, "Scheduled Task Startup Script"
(`T1053_005_OnLogon` and `T1053_005_OnStartup`, both wired to run
`cmd.exe /c calc.exe`), executed via Invoke-AtomicTest against the lab
Windows Server VM.
**Date validated:** 2026-06-22

**Compiled SPL** (pipeline: `splunk_windows`):
```
Image="*\\schtasks.exe" CommandLine="*/create*" CommandLine IN ("*cmd.exe*", "*powershell.exe*", "*pwsh.exe*", "*wscript.exe*", "*cscript.exe*", "*mshta.exe*")
```

**Target corpus (recall):** 2/2 — both scheduled task creation events
detected.

**Benign baseline (false positives):** 0/0 — total `schtasks.exe` activity
in the index at validation time was exactly 2 events, both belonging to the
atomic test itself. There was no pre-existing legitimate scheduled-task
activity to test against.

**Known limitation (logged honestly, not glossed over):** this VM is a
fresh, single-purpose lab image with no Windows Update history, OEM
software, or admin tooling, so it has not yet generated the kind of
background scheduled-task noise a real enterprise endpoint accumulates over
time. The 0% false-positive result reflects an absence of test data, not a
proven-clean result under real-world load. Revisit this rule's precision
once the lab has accumulated more organic background activity, or once a
synthetic-but-realistic benign corpus (e.g. from OTRF Security-Datasets) is
incorporated.

**Cross-domain corpus:** Not yet tested — same reason as above.

**Verdict:** Manually validated against available data, with the precision
caveat above explicitly noted. Promoted to `rules/production/` on the
strength of clean recall and zero observed false positives, pending
re-validation once a richer benign baseline exists.
